## 2026-03-13: 关于看板(Dashboard)部署仓库的讨论

### 用户提问
> "我创建了部署面板的仓库 `https://github.com/Zhang-cm/dashboard-src`，是不应该部署到我笔记的仓库呢？这样会不会影响我的笔记呢？"

### 决策建议：建议分离部署
**结论：建议将看板代码放在独立仓库 `dashboard-src` 中，而不是笔记仓库 `testnote`。**

#### 1. 为什么不建议放在笔记仓库？
- **内容污染**：笔记仓库应该只存放您的知识库（`.md` 文件和图片）。如果混入看板的源代码（`node_modules`, `package.json`, `src/` 等），会导致您的 Obsidian 库变得非常臃肿，且这些代码文件会被 Obsidian 扫描。
- **安全性**：笔记仓库通常是私有（Private）的，而看板由于要发布到 GitHub Pages，通常需要公开（Public）其生成的静态页面。分开存储能更清晰地管理权限。
- **逻辑解耦**：笔记仓库是“数据层”，看板仓库是“展示层”。

#### 2. 是否会影响笔记？
- **不会**。看板在运行时是通过您提供的 **GitHub Token** 动态去读取笔记仓库里的 `.obsidian/sync-stats.json`。
- 只要看板的代码里配置了正确的 `owner` 和 `repo`（即 `Zhang-cm/testnote`），它就能跨仓库读取数据，完全不需要把代码放进笔记文件夹里。

### 下一步执行计划 (Action Items)

1. **修正远程仓库地址**：
   将 `dashboard` 目录下的远程关联修改为新的看板仓库：
   ```bash
   cd dashboard
   git remote set-url origin https://github.com/Zhang-cm/dashboard-src.git
   ```

2. **重新推送代码**：
   将看板源码推送到 `dashboard-src` 仓库的 `main` 分支（或 `dashboard-src` 分支）：
   ```bash
   git push origin dashboard-src -f
   ```

3. **配置 GitHub Pages**：
   在 `Zhang-cm/dashboard-src` 仓库的 Settings -> Pages 中开启服务。
