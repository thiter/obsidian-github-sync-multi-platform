# Obsidian-Github-Sync-Multi-Platform 部署与测试指南

本指南将协助您完成 **Obsidian-Github-Sync-Multi-Platform 插件**的配置以及 **数据看板 (Dashboard)** 的 GitHub Pages 部署。

---

## 1. 前置准备

1. **GitHub 仓库**：创建一个私有仓库（例如 `my-notes`）用于存储笔记。
2. **GitHub PAT (令牌)**：
   - 访问 [GitHub Settings -> Tokens](https://github.com/settings/tokens)。
   - 生成一个具有 `repo` 权限的 **Fine-grained token** 或 **Classic token**。
   - **记录下这个 Token**，稍后在插件和看板中都需要用到。

---

## 2. 插件端部署 (Obsidian)

### 2.1 编译与安装
1. 进入插件目录：`new-obsidian-github-sync-multi-platform-master`。
2. 运行终端并执行：
   ```bash
   npm install
   npm run build
   ```
3. 将生成的 `main.js`、`manifest.json`、`styles.css` 复制到您的库目录：
   `.obsidian/plugins/github-sync/`。

### 2.2 插件配置
1. 在 Obsidian 中启用插件。
2. 进入插件设置页，填写以下信息：
   - **GitHub Owner**: 您的 GitHub 用户名。
   - **GitHub Repo**: 笔记仓库名（如 `my-notes`）。
   - **GitHub Branch**: 默认为 `main`。
   - **GitHub Token**: 填写刚才生成的 PAT。
3. 点击“同步全部笔记”进行首次初始化。

---

## 3. 看板端部署 (GitHub Pages)

我们推荐使用 GitHub Actions 自动部署看板。

### 3.1 准备看板代码
1. 将 `dashboard` 文件夹下的内容提交到您的笔记仓库的一个独立分支（例如 `dashboard-src`），或者建立一个专门的仓库。
2. 在该仓库中，确保 `vite.config.ts` 中的 `base` 设置为 `'./'`。

### 3.2 自动化部署 (推荐)
在仓库中创建文件 `.github/workflows/deploy-dashboard.yml`：

```yaml
name: Deploy Dashboard
on:
  push:
    branches: [ "dashboard-src" ] # 当此分支有变动时触发
jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
      - name: Install and Build
        run: |
          cd dashboard
          npm install
          npm run build
      - name: Deploy to GitHub Pages
        uses: JamesIves/github-pages-deploy-action@v4
        with:
          folder: dashboard/dist
          branch: gh-pages # 部署到 gh-pages 分支
```

### 3.3 启用 Pages
1. 在 GitHub 仓库设置中，进入 **Pages** 选项。
2. 在 **Build and deployment** 下，选择 **Deploy from a branch**。
3. 选择 `gh-pages` 分支和 `/ (root)` 目录，保存。
4. 稍等片刻，GitHub 会提供一个 URL（如 `https://user.github.io/my-notes/`）。

---

## 4. 测试步骤

### 4.1 插件同步测试
1. **新建笔记**：在 Obsidian 中新建一个 `.md` 文件，输入内容。
2. **观察日志**：打开 Obsidian 控制台 (`Ctrl+Shift+I`)，确认是否有 `Synced to GitHub` 字样。
3. **验证远端**：访问 GitHub 仓库，确认文件是否已上传。
4. **验证统计**：检查仓库中是否自动生成了 `.obsidian/sync-stats.json` 文件。

### 4.2 看板功能测试
1. **访问看板**：打开 GitHub Pages 提供的 URL。
2. **初始化配置**：
   - 首次打开会显示配置页，填入 Owner, Repo 和 Token。
   - 设置您的 Dashboard 登录密码和找回邮箱。
3. **登录验证**：使用刚才设置的密码登录。
4. **数据同步**：
   - 点击看板上的刷新按钮。
   - 确认图表是否正确显示了本月的字数统计。
5. **异常测试**：修改笔记后等待 5 秒，再次刷新看板，确认字数是否增长。

---

## 5. 常见问题 (FAQ)

- **看板显示连接失败**：请检查 PAT 权限是否包含 `repo` 且仓库名填写正确。
- **字数不更新**：插件有 5 秒防抖逻辑，请在修改后等待片刻再查看远端。
- **忘记看板密码**：访问看板页，点击 "Forgot Password?"，系统会提示您的找回邮箱。如需重置，请手动删除 GitHub 仓库中的 `.obsidian/auth.json`。
